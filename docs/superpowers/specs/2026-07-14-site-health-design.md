# 后台站点健康检查设计

## 目标

在现有管理后台增加一个只读的站点健康检查，集中发现内容元数据、源文件、生成产物和媒体文件之间的不一致。检查结果只用于诊断，不自动删除、改写或重建任何文件。

本功能面向本地管理员，不出现在公开网站，也不引入数据库、后台任务或外部监控服务。

## 架构

新增 `backend/site_health.py` 作为纯检查服务。该模块只接收项目路径、数据和密码状态等依赖，返回结构化结果，不导入 Flask，也不直接生成 HTTP 响应。

新增 `backend/routes/health.py`，提供受现有 session 守卫保护的 `GET /api/site-health`。路由调用检查服务并返回：

```json
{
  "status": "healthy|warning|error",
  "summary": {"passed": 0, "warnings": 0, "errors": 0},
  "checks": [
    {
      "id": "photos.variants",
      "label": "照片文件完整性",
      "status": "passed|warning|error",
      "severity": "info|warning|error",
      "message": "检查摘要",
      "details": ["具体问题"]
    }
  ]
}
```

顶层状态由检查项聚合：存在 `error` 时为 `error`，否则存在 `warning` 时为 `warning`，其余为 `healthy`。检查项按错误、警告、正常排序，同级保持固定顺序。

## 首批检查范围

1. 数据文件：后台使用的 JSON 文件存在、可以解析，且顶层类型符合预期。
2. 随笔一致性：每条随笔都有合法 slug、对应 Markdown；本地已构建时应有对应 HTML，缺失生成产物记为警告而不是错误。
3. 加密随笔：设置密码的随笔应有 Markdown 密文和包含密码门的 HTML；密码状态与密文状态不一致记为错误。
4. 照片完整性：`photos.json` 中的照片应有原图及 `lg/md/sm` 变体；缺失任一引用文件记为错误。
5. 照片故事：封面和照片列表必须引用现有照片。
6. 音乐与轨迹：JSON 记录引用的 MP3、GPX 文件必须存在。
7. 孤立文件：照片变体、音乐和轨迹目录中未被数据引用的文件记为警告；头像、随笔配图和维护临时文件不纳入孤立媒体判断。
8. 外链协议：Work、Contact、Friends 的非空 URL 只允许 `http://` 或 `https://`。
9. 构建依赖：核心模板、首页脚本和样式文件必须存在。

检查不请求外部 URL，因此“外链健康”只验证协议和结构，不判断远程网站是否在线。所有路径比较使用规范化后的绝对路径，检查逻辑不得读取项目目录之外的文件。

## 后台界面

`admin.html` 新增独立 `Health` 标签页，并加载 `assets/js/admin-health.js`。界面保持现有轻量编辑风格，不使用图表或复杂仪表盘。

页面包含：

- 总体状态与错误、警告、通过数量。
- “重新检查”按钮；不自动轮询。
- 按严重程度分组的检查项列表。
- 每项可展开具体文件或记录信息；所有内容通过 DOM `textContent` 写入，不拼接未经转义的 HTML。

Dashboard 增加一个紧凑健康摘要。Dashboard 和 Health 标签页都请求同一接口；不新增第二套统计模型。健康检查失败时只显示局部错误，不影响其他后台标签页。

## 错误处理与性能

单个检查项捕获自己的可预期文件或解析错误，并将问题转为结构化结果；一个损坏 JSON 不应阻止其他目录检查。只有检查服务自身发生未预期异常时，接口才返回 500。

检查在请求内同步执行。当前内容规模较小，不做缓存和后台任务；目录扫描限定在 `data/`、`md/`、`essays/`、`images/`、`music/`、`tracks/` 和核心模板资源。结果不得包含密码、密文正文或其他秘密值。

## 测试

- 单元测试使用临时目录覆盖正常、缺失文件、损坏 JSON、加密状态不一致、孤立媒体和危险 URL。
- 路由测试覆盖未登录拒绝、正常返回、聚合状态和未预期异常的 500 响应。
- 浏览器烟测覆盖 Health 标签切换、摘要渲染、刷新按钮和 Dashboard 摘要。
- 完整运行 pytest、全部 JavaScript 语法检查、Python 编译检查和 Graphify 增量更新。

## 非目标

- 自动修复、删除孤立文件或重建站点。
- 外部 URL 可达性检查、定时监控或通知。
- GitHub Pages 线上诊断。
- 历史健康趋势和持久化扫描结果。
- 替代现有 pytest、构建流程或 Git 工具。
