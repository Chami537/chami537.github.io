# 后台内容统计设计

## 目标

在现有管理后台首页增加一个轻量的内容概览，帮助管理员快速了解网站当前内容规模和最近更新情况。

本次只做当前总量、标签分布和最近更新，不做趋势图、历史统计、独立仪表盘页面或公开展示。

## 后端接口

新增 `GET /api/dashboard-stats`，沿用现有后台 session 鉴权。接口从现有 JSON 数据和随笔数据读取内容，不新增数据库，也不改变现有 CRUD 接口。

返回结构固定为：

```json
{
  "counts": {
    "essays": {"total": 0, "public": 0, "hidden": 0, "encrypted": 0},
    "photos": 0,
    "photo_stories": 0,
    "places": 0,
    "work": 0,
    "music": 0,
    "friends": 0,
    "stack": 0
  },
  "tags": [{"name": "技术", "count": 0}],
  "recent": [{"type": "essay", "title": "...", "date": "...", "url": "..."}]
}
```

文章统计以现有 hidden/password 语义为准：`total` 包含所有文章，`public` 为公开文章，`hidden` 为隐藏文章，`encrypted` 为设置密码的文章。标签统计只统计文章标签，按数量降序排列；同数量时按标签名稳定排序。最近更新合并随笔和照片故事，按已有日期字段降序返回有限条目，并保留可打开的页面地址。

如果读取数据失败，接口返回适当的 5xx 错误和简洁错误信息，不伪造部分成功结果。

## 后台界面

在 `admin.html` 现有首页内容区域加入概览模块，保持当前管理面板的布局和视觉规范，不新增独立路由。

模块包含三组信息：

1. 内容数量：随笔、照片、照片故事、地点、作品、音乐、朋友、技术栈。
2. 随笔状态：总数、公开、隐藏、加密。
3. 标签与最近更新：展示高频标签和最近更新的随笔/照片故事，可点击打开对应页面。

页面加载时请求统计接口。加载期间显示轻量占位状态；失败时显示可理解的错误提示，并保留后台其他标签页和操作。统计模块不主动轮询，内容变更后由现有页面刷新机制或重新进入首页触发更新。

## 数据与边界

- 复用现有 `backend/data.py` 路径常量和 JSON 读取工具。
- 复用现有鉴权装饰器或守卫，统计接口不得在未登录状态下暴露内容摘要。
- 不缓存统计结果，确保管理员进入首页时看到最新数据。
- 不修改生成的静态首页，也不把管理员统计写入公开构建产物。

## 测试

- 后端测试覆盖已登录访问、未登录拒绝、正常聚合、标签排序、最近更新排序和数据读取失败。
- 前端测试或静态检查覆盖统计模块的请求、成功渲染和失败状态。
- 运行完整 pytest，并检查新增 JavaScript 语法。

## 非目标

- 按月新增趋势和历史快照。
- 访客分析、访问量、来源统计。
- 草稿工作流、批量编辑和发布检查。
- 公开页面展示统计数据。
